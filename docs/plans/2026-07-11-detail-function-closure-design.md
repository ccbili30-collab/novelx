# NovelX 细节功能闭环设计

## 目标

本轮暂停扩展新阶段，集中修复 Agent（智能体）工作台的基础交互：当前项目默认可用、空项目自然反馈、最后一句撤回重发、消息复制、审查状态真实性和真实 Provider（提供方）用户旅程。

## 项目范围

Renderer（渲染进程）可以表达用户当前选中的资源，但不能决定最终授权。WorkspaceSession（工作区会话）从当前工作区数据库读取全部有效资源，向 AgentProcessSupervisor（智能体进程监督器）提供后端权威范围。显式选中资源时必须属于该范围；没有显式选择时，默认使用项目中的创作对象，空项目则使用领域根节点，使 Agent 能检索后得出“暂无内容”，而不是“没有授权”。

## 最后一句撤回重发

只允许撤回当前会话最后一条用户消息以及它后面的 assistant/error（助手/错误）回复。数据库使用单个事务检查并删除这段尾部。会话正在运行，或尾部包含 pending/committed Change Set（待审查/已提交变更集）时拒绝撤回，避免聊天历史和项目状态分离。成功后原文本回填输入框，由用户修改并重新发送。

## 消息操作

用户消息和 Agent 回复都提供复制按钮；只有最后一条可安全撤回的用户消息显示编辑按钮。按钮使用图标和 tooltip（悬停提示），复制成功显示短暂完成状态。空的待审查区域不渲染，只有真实待审查变更或载入失败才占用界面。

## 验证

- Repository（仓库）事务单元测试覆盖普通撤回、项目副作用阻塞和运行中阻塞。
- Supervisor（监督器）测试覆盖后端默认范围与越界拒绝。
- Playwright（界面自动化）覆盖复制、撤回重发、空审查区隐藏。
- 使用本机加密 Provider 配置的隔离用户目录完成真实模型空项目和已有资料项目用户旅程。
