# NovelX 0.2.3 细节功能闭环

日期：2026-07-11

## 修复原因

- 空项目只有领域根节点，Renderer（渲染进程）过去过滤这些节点后传递空范围，Agent（智能体）将其错误理解为“没有授权”。
- 项目范围主要由前端传递，Main（主进程）没有用当前工作区数据库进行最终校验。
- 会话消息没有单条尾部撤回事务，因此无法安全实现“修改上一句”。
- 消息没有复制操作。
- Pending Change Sets（待审查变更集）为空时仍显示“待审查 0”，容易被理解成当前回复需要审查。
- 第一版消息操作栏覆盖短文本，自动化功能通过但视觉不合格。

## 已实现

- WorkspaceSession（工作区会话）从当前工作区数据库生成权威资源范围。
- 没有显式选择时，普通项目默认覆盖当前创作对象；空项目回退到六类领域根节点。
- AgentProcessSupervisor（智能体进程监督器）拒绝 Renderer 提交的跨项目资源范围。
- 空项目真实检索后自然说明“当前没有具体内容”，不再要求用户提供项目名称或内部资源 ID。
- 用户和 Agent 消息均支持复制。
- 只有当前会话最后一条用户消息支持“修改上一句”。
- 修改上一句会在数据库事务中撤回最后一条用户消息和它后面的回复，并把原文放回输入框。
- 会话正在运行，或回复已经产生 pending/committed Change Set（待审查/已提交变更集）时拒绝撤回。
- 没有真实待审查变更时不显示待审查区域；切换工作区时先清空旧列表。
- 消息操作栏经过 Playwright（界面自动化）截图检查，最终放在用户气泡左侧，不覆盖正文。

## 验证

- TypeScript（类型脚本）类型检查通过。
- Vitest（单元测试框架）：74 个文件、285 项测试通过。
- Playwright 完整 Electron（桌面应用）回归：32 项通过、4 项因缺少专用真实模型环境跳过。
- 隔离真实 Provider（提供方）旅程：复制本机加密 Provider 配置和同目录 Local State 到临时用户目录，真实调用 deepseek-v4-flash，空项目回复通过。
- 真实旅程确认用户回复不包含“没有获得任何已授权”“授权范围”或“资源 ID”，且不显示空待审查区。

## 明确未完成

- 这不是全产品所有细节的最终精修，只是第一轮 Agent 工作台基础交互闭环。
- 只支持修改最后一句，不支持编辑任意历史消息或对话分支。
- 已经产生项目变更的消息不能直接撤回；需要后续与项目检查点回溯联动。
- 本轮没有重新设计右侧活动区、IDE 编辑器、导入流程或玩家模式视觉。
- Windows 安装包仍没有受信任的代码签名证书。
