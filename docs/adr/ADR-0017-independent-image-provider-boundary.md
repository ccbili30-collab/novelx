# ADR-0017：独立图片 Provider 与 Responses 生图边界

日期：2026-07-13
状态：Accepted（已接受）

## 背景

黑客松主路径要求 NovelX 调用真实图片模型生成角色或场景图片。文本 Provider（模型服务）已有独立配置和凭据，但图片服务可能使用不同代理、模型、费用和协议。用户已授权第一天增加独立图片模型设置，并提供一份仅写入本机安全存储的测试配置。

实际探测证明当前代理支持 OpenAI-compatible Responses API（兼容响应接口）与 `image_generation` 工具；因此不能按尚未验证的 `/images/generations` 接口实现。

## 决策

1. 图片 Provider 使用独立配置、独立凭据和独立 IPC（进程间通信）命名空间，不复用文本 Provider 凭据。
2. 第一版协议固定为 `POST /v1/responses` 与 `image_generation`，不建立未经需要证明的通用多协议框架。
3. API Key（接口密钥）只允许进入 Electron Main Process（主进程）与 `safeStorage`；Renderer（渲染进程）、工作区、日志和公开状态只能看到 `hasCredential`。
4. Base URL（服务地址）必须使用 HTTPS；HTTP 仅允许精确本地回环主机。
5. 真实连接测试会产生一次最小图片生成费用；界面必须明确提示。成功结果只返回模型、耗时、MIME、像素、字节数和 SHA-256，不持久化测试图片。
6. 只接受响应内的 Base64 图片；第一版不跟随模型返回的任意图片 URL，避免把图片服务响应变成 SSRF（服务端请求伪造）入口。
7. 未配置凭据、系统安全存储不可用、网络失败或响应非法时 Fail Closed（失败关闭），不得用占位图或本地模板冒充真实生图。

## 后果

- 当前桌面设置页可以安全保存、替换、清除并真实测试独立图片模型。
- 该决策只完成 Provider 配置与连通性边界；图片 Job（任务）、Asset（资产）、大管家工具和联合展台仍需后续任务实现。
- 若后续服务只支持其他图片协议，必须新增明确适配器和对应真实验收，不修改本 ADR 来伪称已有兼容。
