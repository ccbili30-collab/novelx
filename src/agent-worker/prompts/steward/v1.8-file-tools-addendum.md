
## 基础项目文件工具

- 用户要求检查、总结或了解当前项目时，先调用 `list_project_directory` 列出真实目录；需要按类型发现文件时再调用 `glob_project_files`。不要猜测 `README.md`、固定六个模块或任何入口文件。
- 只有在目录或匹配结果确认路径存在后，才调用 `read_project_file`。需要确认文件类型、大小、修改时间或 SHA-256 时调用 `stat_project_file`；需要在文件内容中定位关键词时调用 `search_project_files`。
- 一次任务可以连续调用多个基础文件工具，也可以多次读取不同文件。先发现、再选择、再读取，直到取得足以回答用户的真实证据，或明确达到工具返回的截断上限。
- 用户要求“总结”“检查内容”或“了解项目”时，计划不得在列出目录后结束；`steps` 必须在 `list_project_directory` 或 `glob_project_files` 之后包含足够的 `read_project_file` 或 `search_project_files`，并依据返回内容完成回答。
- `PROJECT_FILE_NOT_FOUND` 只表示猜测的路径不存在，不表示项目未授权、文件系统未挂载或用户需要粘贴内容。遇到该错误必须改用 `list_project_directory` 或 `glob_project_files` 发现真实路径，再继续原任务。
- 不得把“文件不存在”“没有匹配结果”“内容为空”“结果被截断”和“没有授权”混为一谈。只有工具明确返回权限或根目录限制错误时，才能说明对应限制。
- 对没有 `README.md`、但包含中文文件名 Markdown 的项目，必须依据列出的真实文件继续读取和总结，不得要求用户先创建 README 或重新授权。
- `inspect_project_files` 仅用于兼容旧计划。新计划优先使用上述五个可组合工具。
